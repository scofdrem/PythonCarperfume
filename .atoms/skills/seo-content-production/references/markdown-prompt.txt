#### 2.2 Article Structure

1. H1 Title
   - Include the primary keyword
   - Keep it compelling and within 60 characters for English; for CJK languages (zh, ja, ko), keep within 25-30 characters

2. Hero Image
   - Format: ![{page_title}]({hero_image_url})
   - Use the provided placeholder exactly as-is; the system will replace it with the final CDN URL later

3. Introduction
   - 200-300 words
   - 2-3 short paragraphs
   - Hook the reader with a pain point or clear promise
   - Use the primary keyword naturally in the first 1-2 sentences
   - State what the article covers

4. Body Sections
   - 4-6 sections, each 250-400 words
   - Follow the provided outline
   - Use descriptive H2s that include primary or secondary keywords when natural
   - Include 2-3 H3 subsections per H2 where useful
   - Explain concepts clearly, provide step-by-step advice when relevant, and use concrete examples
   - Use bullet lists, numbered lists, comparison blocks, or tables when they improve clarity

5. FAQ Section
   - Optional but recommended
   - Add 3-5 FAQ entries when relevant
   - Use real-search style questions with concise answers

6. Conclusion
   - 150-200 words
   - 2-3 short paragraphs
   - Summarize takeaways, reinforce the benefit, and end with a clear CTA

#### 2.3 On-Page SEO Checklist

Ensure the primary keyword appears in:
- H1 title
- Meta description
- First 100 words of the introduction
- At least one H2 heading
- Conclusion

Use secondary keywords naturally throughout the body. Avoid keyword stuffing.

#### 2.4 Internal Linking

- Include 3-5 internal links from the candidate list
- Place links naturally within relevant paragraphs
- Use descriptive anchor text based on the linked page title
- Format: [Page Title](/blog/url-path)

#### 2.5 CTA Integration

If `project_context.routes` contains CTA routes such as `/signup`, `/pricing`, `/demo`, or `/download`:
- Add 1-2 CTA links naturally in the middle sections and/or conclusion
- Use action-oriented anchor text appropriate for {lang}
- Format: [CTA Text](route_path)

If no CTA routes exist, skip CTA links.

### 3. Style and Formatting Rules

- Aim for an 8th-10th grade reading level with simple, direct language (for non-English languages, adapt to an equivalent plain-language standard appropriate for the target language)
- Prefer active voice (for languages where passive is more natural in formal writing, follow local conventions)
- Keep paragraphs short (2-4 sentences each)
- Remove fluff; every paragraph should add value
- Use smooth transitions between sections
- Write like an expert helping a smart but non-expert reader
- Total length should be 1500-2500 words
- Do NOT explain what you are doing
- Do NOT wrap the output in code blocks
- Output the complete publication-ready markdown article directly
